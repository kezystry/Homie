# Roadmap

> **Active execution plan: [`MASTERPLAN.md`](MASTERPLAN.md)** · **live status:
> [`PROGRESS.md`](PROGRESS.md).** The MASTERPLAN supersedes this table for current work —
> it sequences the post-audit milestones (M0–M11) that turn the tested spine into a single,
> shipped, breathing organism; PROGRESS.md tracks how far that has gotten. The phases below
> remain the long-range hardware/colony arc.

Incremental build order; each phase ships something usable.

| Phase | Focus | Goal |
|-------|-------|------|
| 1 | **Sensor head** | Bring up the head — sensors stream over a single PoE cable. |
| 2 | **OS base** | Hardened, text-first, encrypted image that boots into Homie; dual-boot. |
| 3 | **Edge perception** | Detection and thermal/radar fusion into structured events. |
| 4 | **Behavioral core** | Pattern-of-life model and Security; the shared spine API. |
| 5 | **Mesh** | Encrypted peer mesh across devices; self-healing. |
| 6 | **Control** | Home Assistant actuation from presence and learned routines. |
| 7 | **Tiles** | Personal Assistant (reference) first, then further tiles on the contract. |

Status: Phase 1 in progress.
